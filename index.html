Step 2: Get Your GitHub Token

Why You Need This:

This lets us show you live, up-to-date open source projects. You'll also use this same token to submit your completion Pull Request at the end of the course!

How to Get It (takes 1 minute):

  1. Don't have a GitHub account? Create one here (free, takes 2 minutes)
  2. Click this link: Create Token
  3. Under "Note", put what the token is for (e.g., "OpenSourceLearningToken")
  4. Expiration: Choose "7 days" or "30 days"
  5. Under "Select scopes", check the box: ✅ public_repo
    • Why? This allows the app to fork public repositories and create pull requests on your behalf
  6. Scroll down and click "Generate token"
  7. Copy the token (starts with "ghp_")
  8. Paste it below

⚠️ Important: Save your token somewhere safe! GitHub only shows it once.

Module 1: Understanding Open Source 🌍

Duration: ~10 minutes | Build foundational knowledge

📖 What is Open Source?

Open source software is code that anyone can view, use, modify, and share. Unlike proprietary software (like Microsoft Word), open source projects make their code publicly available.

Why it matters:

  • Powers the internet: Most websites run on open source (Linux, Apache, etc.)
  • Free to use: Anyone can use it without paying
  • Community-driven: Thousands of developers collaborate globally
  • Career benefits: Contributing builds your portfolio and network

Popular Open Source Licenses:

  • MIT: Very permissive - do almost anything with the code
  • GPL: Must share your modifications
  • Apache 2.0: Similar to MIT but with patent protection

Module 2: Finding Your First Issue 🔍

Duration: ~15 minutes | Learn to identify good contributions

📖 Understanding GitHub Labels

GitHub uses labels to organize issues. Here are the most important ones for beginners:

  • "good first issue": Perfect for newcomers
  • "beginner-friendly": Doesn't require deep knowledge
  • "documentation": Help improve docs (great for beginners!)
  • "help wanted": Maintainers are looking for contributors

What Makes a Good First Issue?

  • ✅ Clear description of what needs to be done
  • ✅ Active project (recent commits)
  • ✅ Friendly maintainers who respond to questions
  • ✅ Good documentation

Red Flags to Avoid:

  • ❌ No activity in 6+ months
  • ❌ Vague issue descriptions
  • ❌ Lots of open PRs with no maintainer response
  • ❌ Issues that require deep knowledge of the codebase

Module 3: Making Your First Contribution 🚀

Duration: ~20 minutes | Step-by-step contribution process

📖 The Git Workflow

Contributing to open source follows these steps:

  1. Fork: Create your own copy of the project
  2. Clone: Download it to your computer
  3. Branch: Create a new branch for your changes
  4. Edit: Make your changes to the code
  5. Commit: Save your changes with a message
  6. Push: Upload your changes to GitHub
  7. Pull Request (PR): Ask the project to include your changes

Writing Good Commit Messages

Good: "Fix typo in README installation section"

Bad: "fixed stuff"

Tips:

  • Start with a verb: Fix, Add, Update, Remove
  • Be specific about what changed
  • Keep it under 50 characters

PR Etiquette

  • ✅ Be polite and professional
  • ✅ Respond to feedback promptly
  • ✅ Make requested changes
  • ✅ Say thank you!

Module 4: Join the Community 🎉

Duration: ~10 minutes | Submit your first real PR!

📖 Open Source Etiquette

  • Be respectful: Remember, maintainers are volunteers
  • Read the guidelines: Check CONTRIBUTING.md before submitting
  • Ask questions: Don't be afraid to ask for help
  • Be patient: Reviews can take days or weeks
  • Give back: Help others once you're comfortable

Continuing Your Journey

After this course, here's what to do:

  • Start with documentation fixes (typos, clarity improvements)
  • Join project Discord/Slack channels
  • Follow projects you're interested in
  • Contribute regularly - even small PRs matter!
